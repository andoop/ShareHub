# 全局红线 · Constraints

> 全局唯一。任何需求、任何预演、任何实现都不得违反。预演中发现违反即为 ANOMALY。

## 绝对必须（MUST）
- [ ] 所有外部输入（上传文件元数据、分享配置、下载请求）必须校验
- [ ] 分享链接与下载必须具备可配置的访问验证（密码和/或令牌），不得默认完全公开
- [ ] 敏感配置（密钥、管理员凭据）不得硬编码入库，使用环境变量或受保护配置
- [ ] 改动必须有对应测试或验收用例覆盖（见各 feature `tests.md`）
- [ ] 核心用户路径必须极简（上传→分享→下载），不得以功能堆叠牺牲易用性
- [ ] 交付质量按商业软件标准：一致 UI、完整状态与错误反馈、中文用户文案、响应式移动端

## 绝对禁止（MUST NOT）
- [ ] 禁止在未明确要求时引入新的重量级第三方依赖
- [ ] 禁止写未被要求的兜底逻辑 / 节外生枝
- [ ] 禁止将用户文件或分享令牌写入公开日志
- [ ] 禁止默认无鉴权暴露上传接口或管理控制台
- [ ] 禁止暴露给用户内部错误码、堆栈或未翻译的技术术语

## 技术约束
- 语言 / 框架 / 版本：待 feature PRD 方案选定后写入 plan.md（当前无既有栈约束）
- 代码风格 / lint 规则：随选定技术栈沿用社区默认（待 plan 固化）
- 性能 / 安全 / 合规要求：大文件传输需支持流式/分块；HTTPS 部署为生产 MUST（自托管由运维配置）
