# 便捷文件分享与同步服务 PRD

> 对应 `project.md` 北极星 / 继承 `constraints.md` 红线。实现细节见 `plan.md`（PRD 确认后编写）。

## 1. 目标
提供一个自托管（或可部署到自有服务器）的**文件分享枢纽**：分享者通过网页控制台上传文件、配置分享策略并生成链接/二维码；接收者扫码或打开链接，经验证后下载——**无需经过微信/企业微信等 IM 中转**。

## 2. 背景与现状
- 项目为绿地，无既有业务代码（来源：2026-06-15 侦察，项目根无 `src/`/`app/` 等应用目录）。
- 用户现状：大文件需登录 IM → 上传 → 对方下载，步骤多、体积受限、不便管理「谁可以下什么」。
- 竞品/参考形态：WeTransfer、奶牛快传、Pingvin Share、FileBrowser——具备网页上传、链接分享、可选密码；本需求强调**自有控制台 + 可控分享策略 + 二维码**。

## 3. 用户故事 / 场景
- 作为**分享者**，我希望在网页上选择文件并一键生成分享链接与二维码，以便手机扫码或发链接给他人直接下载。
- 作为**分享者**，我希望在控制台看到所有分享项、可单独设置密码/有效期/撤销，以便精细控制「哪些文件、怎么分享、分享给谁」。
- 作为**接收者**，我希望打开链接或扫码后输入提取码（若需要）即可下载，以便无需安装 IM 或注册复杂账号。
- 作为**分享者（自己）**，我希望同一服务能在电脑上传、手机扫码下载，以便**设备间同步**个人文件。

## 4. 方案探索

| 维度 | 方案 A：集成成熟开源 | 方案 B：轻量自研全栈 ⭐推荐 | 方案 C：Serverless + 对象存储 |
|------|---------------------|---------------------------|------------------------------|
| 形态 | 部署 Pingvin Share / FileBrowser 等，按需改皮肤与配置 | 单体 Web 应用：API + 管理台 + 公开下载页 | 前端 + Lambda/云函数 + S3/OSS 预签名 URL |
| 优点 | 最快可用、功能现成 | 完全贴合「控制台 + 验证 + 二维码」需求；代码量可控 | 弹性好、存储专业 |
| 缺点 | 定制「分享给谁」等策略受限于上游；升级耦合 | 需自研安全与存储逻辑 | 云厂商绑定、本地/NAS 场景不友好 |
| 适合 | 先验证需求、接受通用产品形态 | **个人/小团队自托管、要强控制** | 公网 SaaS、有云服务预算 |

**推荐方案 B**，理由：用户明确要「控制台分别控制哪些文件怎么分享、分享给谁」，绿地项目无历史包袱，轻量自研可精确实现验证模型与二维码流程，且可部署在 NAS/VPS。若 Q3 确认倾向「尽快能用」，可降级为方案 A 并收窄首期 MUST。

## 5. 功能需求

### 5.1 分享者控制台（需登录）
- FR1: 分享者使用账号登录管理控制台。〔已确认: Q2 选 A，首期单管理员账号 + 环境变量配置，无接收者账号体系〕
- FR2: 上传一个或多个文件，展示文件名、大小、上传时间。〔已确认: 用户原始需求〕
- FR3: 为每个文件（或文件包）创建**分享项**，可配置：是否启用、提取码（可选）、过期时间（可选）、最大下载次数（可选）。〔已确认: 用户「验证机制」「控制怎么分享」〕
- FR4: 分享项生成**唯一分享链接**与**二维码**（指向公开下载页 URL）。〔已确认: 用户「生成二维码，方便手机扫码」〕
- FR5: 控制台可列表查看、编辑、撤销分享项；撤销后链接立即失效。〔已确认: 用户「分别控制着哪些文件怎么分享」〕
- FR6: 可选：为分享项标注接收者备注/标签，便于分享者自行管理「发给谁」（首期不做强制身份校验名单）。〔已确认: Q2 选 A，仅备注字段，不做名单校验〕

### 5.2 公开下载页（无需控制台账号）
- FR7: 访问分享链接打开下载页，展示文件基本信息（名称、大小；不暴露存储路径）。〔已确认〕
- FR8: 若分享项启用提取码，用户须输入正确提取码后才能下载。〔已确认: 用户「需要验证的机制」〕
- FR9: 过期/撤销/超次数的分享返回明确错误页，不泄露文件是否存在。〔已确认: 安全基线〕
- FR10: 下载支持浏览器直链下载；大文件采用流式响应，避免内存撑爆。〔已确认: 用户「东西很大」〕

### 5.3 设备间同步（同一分享者）
- FR11: 分享者可将文件分享给自己：生成链接/二维码，在手机浏览器打开并下载，实现跨设备同步。〔已确认: 用户「给自己手机同步东西」〕

### 5.5 易用性与商业级体验
- FR14: **核心路径极简**——分享者从登录到「文件已可分享」≤ 3 步（选文件 → 上传 → 生成分享）；接收者扫码/开链接 → 输入提取码（若有）→ 下载，全程无需读文档。〔已确认: 手机反馈「一定要方便使用」〕
- FR15: 控制台与下载页均为**响应式设计**，手机浏览器与桌面同等可用（大按钮、单手操作友好）。〔已确认: 方便使用 + 设备间同步场景〕
- FR16: 所有用户可见文案为**中文、直白**；错误页说明原因与下一步（如「提取码错误，请向分享者确认」），禁止技术堆栈信息。〔商业软件标准〕
- FR17: 上传与下载均有**进度反馈**（百分比或 determinate 指示）；大文件传输中页面保持可响应，不无声挂起。〔商业软件标准〕
- FR18: 视觉与交互达到**商业软件水准**：统一设计系统、加载态/空状态/成功态完整、表单校验即时提示、关键操作可撤销确认。〔已确认: 手机反馈「按商业软件要求来做」〕

- FR12: 提供 Docker Compose 一键部署，数据目录与配置可持久化；默认绑定局域网地址，文档说明 NAS/本机访问方式。〔已确认: Q1 选 NAS/内网〕
- FR13: 生产环境文档要求 HTTPS 反向代理（Caddy/Nginx），不在应用内硬编码证书。〔继承: constraints.md〕

## 6. 验收标准（成功定义 · 抽象层）
> 具体场景见 `tests.md`（PRD 确认后编写）。

- [ ] 分享者能在控制台上传文件，并为该文件生成带可选提取码的分享链接与可扫描的二维码
- [ ] 接收者通过链接或扫码，在提取码正确（若启用）且分享未过期/未撤销时，能成功下载完整文件
- [ ] 分享者能在控制台撤销或修改分享策略，变更对后续访问立即生效
- [ ] 未授权访问（错误提取码、过期链接、已撤销链接）被拒绝，且不泄露敏感路径信息
- [ ] 分享者可用同一流程完成「电脑 → 手机」自我同步，无需 IM 中转
- [ ] 核心分享路径步骤极少、手机端可用，非技术用户无需阅读文档即可完成首次上传与分享
- [ ] 界面与交互达到商业软件水准：一致视觉、完整状态反馈、清晰中文错误提示

## 7. MUST（绝对要做）
- **易用性是首要产品目标**——任何实现不得牺牲「上传→分享→下载」路径的简洁性
- **商业软件交付标准**：完整错误处理、无粗糙占位 UI、关键流程可向非技术用户演示
- 网页控制台 + 公开下载页双端分离
- 分享链接 + 二维码生成
- 可配置的分享验证（至少支持提取码；Q2 确认后扩展名单）
- 分享生命周期管理（创建 / 列表 / 撤销 / 过期）
- 继承 `constraints.md` 全部 MUST

## 8. MUST NOT（绝对不能做）
- 不做未被要求的兜底逻辑 / 不节外生枝
- 首期不做 IM 机器人、企业微信插件
- 首期不做多用户 SaaS 计费与复杂 RBAC（除非 Q1/Q2 明确要求）
- 首期不做在线预览/协作编辑/版本历史
- 默认不开放无鉴权上传接口
- 继承 `constraints.md` 全部 MUST NOT

## 9. 非目标 / 暂不做（YAGNI）
- 原生移动 App（可用 PWA 响应式网页代替）
- 端到端加密（首期传输依赖 HTTPS；存储加密可作为后续增强）
- 公共文件搜索引擎、匿名投稿墙
- 与微信/企业微信 API 集成
