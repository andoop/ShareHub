# 测试用例 · 2026-06-15-file-share-hub

> 理解闸门：开发者先读本文件判断 AI 是否懂需求。每条映射回 PRD FR / §6 验收 / MUST。

## TC1 · 控制台登录成功
- **映射**: FR1、验收「分享者能在控制台上传文件」
- **Given**: 服务已部署；环境变量已配置管理员用户名 `admin`、密码 `Secret123!`
- **When**: 分享者在浏览器打开 `http://192.168.1.10:8080/admin`，输入上述凭据并提交
- **Then**: 进入控制台首页，显示已登录状态（如顶部显示「admin」或「已登录」），未登录不可见上传区域
- **状态**: 待验证

## TC2 · 上传文件并出现在列表
- **映射**: FR2、验收「分享者能在控制台上传文件」
- **Given**: 分享者已登录控制台
- **When**: 选择本地文件 `vacation.mp4`（大小 50MB）点击上传
- **Then**: 列表新增一行：文件名 `vacation.mp4`、大小显示约 `50 MB`、状态为「已上传」或等价文案
- **状态**: 待验证

## TC3 · 创建带提取码的分享并生成链接与二维码
- **映射**: FR3、FR4、验收「生成带可选提取码的分享链接与可扫描二维码」
- **Given**: 文件 `vacation.mp4` 已在列表中
- **When**: 点击「创建分享」，启用提取码并设为 `blue7`，保存
- **Then**: 展示分享链接形如 `http://192.168.1.10:8080/s/<token>`（`<token>` 为非空随机串）；同时展示可扫描的二维码图片，扫码 URL 与链接一致
- **状态**: 待验证

## TC4 · 无提取码分享可直接下载
- **映射**: FR8、FR10、验收「接收者…能成功下载完整文件」
- **Given**: 分享项 `report.pdf` 未启用提取码，链接为 `http://192.168.1.10:8080/s/abc123`
- **When**: 接收者在手机浏览器打开该链接，点击下载
- **Then**: 浏览器开始下载 `report.pdf`；下载完成后文件大小与源文件一致（可用校验和或字节数核对）
- **状态**: 待验证

## TC5 · 错误提取码被拒绝
- **映射**: FR8、验收「未授权访问…被拒绝」、MUST「分享必须具备可配置访问验证」
- **Given**: 分享项启用提取码 `blue7`，链接 `http://192.168.1.10:8080/s/xyz789`
- **When**: 接收者打开链接，输入提取码 `wrong1` 并提交
- **Then**: 页面显示「提取码错误」或等价提示；**不**开始下载；HTTP 状态为 401 或 403（实现层可验证，黑盒以页面提示为准）
- **状态**: 待验证

## TC6 · 正确提取码后下载成功
- **映射**: FR8、FR10
- **Given**: 同 TC5 分享项，提取码为 `blue7`
- **When**: 接收者输入提取码 `blue7` 并提交，再点击下载
- **Then**: 文件完整下载；页面不显示存储绝对路径或服务器内部目录
- **状态**: 待验证

## TC7 · 撤销分享后链接失效
- **映射**: FR5、验收「撤销或修改分享策略，变更对后续访问立即生效」
- **Given**: 有效分享链接 `http://192.168.1.10:8080/s/revoke01`
- **When**: 分享者在控制台点击该分享的「撤销」
- **Then**: 控制台该分享状态变为「已撤销」；接收者再次访问链接看到「分享已失效」类页面，无法下载
- **状态**: 待验证

## TC8 · 过期分享被拒绝
- **映射**: FR3、FR9
- **Given**: 分享项创建时设置过期时间为 1 分钟后；当前时间已超过过期时间
- **When**: 接收者访问该分享链接
- **Then**: 显示「分享已过期」；不泄露文件是否存在以外的内部信息
- **状态**: 待验证

## TC9 · 设备间自我同步（电脑→手机）
- **映射**: FR11、验收「分享者可用同一流程完成电脑→手机自我同步」
- **Given**: 分享者在电脑控制台上传 `notes.txt` 并创建无提取码分享；手机与电脑在同一局域网
- **When**: 分享者在电脑查看二维码，用手机浏览器扫码打开并下载
- **Then**: 手机获得 `notes.txt`，内容与电脑源文件一致
- **状态**: 待验证

## TC10 · 未登录无法访问控制台 API
- **映射**: MUST NOT「默认不开放无鉴权上传接口」、constraints「管理控制台须鉴权」
- **Given**: 未携带有效会话/令牌
- **When**: 直接向 `POST /api/files/upload` 发起上传请求
- **Then**: 请求被拒绝（401/403）；文件未入库
- **状态**: 待验证

## TC11 · 分享备注便于管理「发给谁」
- **映射**: FR6
- **Given**: 分享者已登录
- **When**: 创建分享时为备注填入「发给小李」
- **Then**: 控制台分享列表显示备注「发给小李」，便于分享者区分不同接收场景
- **状态**: 待验证

## TC12 · Docker 部署后数据持久化
- **映射**: FR12
- **Given**: 使用 `docker compose up` 首次启动并上传文件；随后 `docker compose down` 再 `up`
- **When**: 分享者重新登录控制台
- **Then**: 此前上传的文件与分享记录仍在列表中（数据卷未丢）
- **状态**: 待验证

## TC13 · 核心路径 ≤3 步完成分享
- **映射**: FR14、验收「核心分享路径步骤极少」
- **Given**: 分享者已登录，控制台为空
- **When**: 从首页开始：选择文件 → 上传完成 → 点击「创建分享」并保存（默认无提取码）
- **Then**: 全程 ≤3 次主要点击/确认；无需跳转帮助页；展示可复制的分享链接
- **状态**: 待验证

## TC14 · 手机端响应式可用
- **映射**: FR15
- **Given**: iPhone Safari 打开 `http://<NAS-IP>:8080/admin` 与 `/s/<token>`
- **When**: 完成登录、上传、下载各一次
- **Then**: 无横向滚动；主要按钮可单手点击；文字可读不溢出
- **状态**: 待验证

## TC15 · 错误提示中文且可操作
- **映射**: FR16、constraints「禁止技术术语」
- **Given**: 访问已撤销分享或输入错误提取码
- **When**: 用户查看错误页
- **Then**: 显示中文说明（如「分享已失效，请联系分享者重新发送」）；不出现 HTTP 状态码、堆栈或英文技术词
- **状态**: 待验证
