# 项目北极星 · Project

> 全局唯一。描述这个项目"为什么存在、要达成什么"。所有需求都服务于此。

## 目标（北极星）
让个人或小团队在不依赖微信/企业微信等 IM 中转的前提下，通过网页与二维码快速、可控地分享与同步文件。

## 背景 / 现状
- 项目根目录当前为**绿地项目**：除 Sandtable 方法论资产与 Mobile Companion runtime 源码外，尚无业务应用代码（来源：项目根 `find` 侦察，2026-06-15）。
- 用户痛点：大文件经社交软件中转需登录、上传再下载，步骤繁琐；缺少统一的分享控制台与访问控制。
- 已安装 Sandtable 工作区：`docs/sandtable/`（来源：`docs/sandtable/project.md` 初始化记录）。

## 范围
- 在范围内：文件上传、受控分享链接/二维码、下载页、分享者控制台、基础访问验证。
- 明确不在范围内（首期）：Flutter 移动端 App（Sandtable `apps/` 不随安装复制）、IM 集成、复杂协作编辑。

## 关键干系人
- 开发者 / 决策人：项目所有者（待确认部署与红线偏好，见 `features/2026-06-15-file-share-hub/questions.md`）
- 目标用户：分享者（自己或给他人发文件的人）、接收者（通过链接/扫码下载的人）
